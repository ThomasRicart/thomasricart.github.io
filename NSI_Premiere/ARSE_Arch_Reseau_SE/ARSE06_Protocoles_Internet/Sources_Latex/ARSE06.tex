\begin{easybox}[Le protocole TCP]{}
	Le protocole TCP (Transmission Control Protocol) est un protocole de la couche 3 (Transport) qui permet de garantir la bonne transmission des données entre les machines. Il utilise un système de numérotation des paquets pour assurer que les données sont reçues dans le bon ordre et qu'aucune donnée n'est perdue.

	Les étapes du processus de handshake sont les suivantes :

	\begin{enumerate}
		\item La machine émettrice envoie un segment SYN (synchronize) pour initier la connexion.
		\item La machine réceptrice répond avec un segment SYN-ACK (synchronize-acknowledge) pour confirmer la réception du segment SYN.
		\item La machine émettrice envoie un segment ACK (acknowledge) pour confirmer la réception du segment SYN-ACK et établir la connexion.
	\end{enumerate}

\begin{methode}[Le bit alterné]{}

    Dans le cas du protocole TCP, l'émetteur doit s'assurer que le message a bien été reçu. 

    En temps normal, l'émetteur (noté E) envoie un message au receveur (noté R) qui répond par un acquittement (ACK pour ACKnowledge) pour confirmer la bonne réception du message. 

    Le protocole du bit alterné prévoit 4 règles:

    \begin{enumerate}
        \item chaque message et son acquittement contiennent un même bit de contrôle ``0'' ou ``1'',
        \item deux messages successifs contiennent des bits de contrôle différents (la valeur du bit de contrôle alterne à
        chaque message),
        \item si E ne reçoit pas l'acquittement ou un acquittement erroné (mauvais bit de contrôle), il renvoie le dernier
        message,
        \item si R reçoit un message avec un bit de contrôle erroné (pas d'alternance ``0'' ou ``1''), il renvoie le dernier
        acquittement.
    \end{enumerate}

    Pour chaque message, un délai doit être respecté pour vérifier les acquittements.
