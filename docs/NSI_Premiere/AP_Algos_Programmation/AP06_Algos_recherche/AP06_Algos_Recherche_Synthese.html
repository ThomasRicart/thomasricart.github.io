AP06 : Algorithmes de Recherche

NSI Première - Complexité, Terminaison et Correction

1. Recherche Naïve et Complexité

La recherche naïve parcourt chaque élément de la liste un par un.

def recherche_naive(elt, liste):
    for x in liste:
        if x == elt:
            return True
    return False
Complexité Temporelle : Dans le pire des cas, on effectue n comparaisons. La complexité est dite linéaire, notée O(n).

2. Terminaison et Correction

Pour prouver qu'un algorithme est fiable, on utilise deux outils théoriques:

  • Le Variant de Boucle : Une quantité entière qui décroît strictement à chaque tour tout en restant minorée (garantit que la boucle s'arrête).
  • L'Invariant de Boucle : Une propriété qui reste vraie avant et après chaque itération (prouve que l'algorithme fait ce qu'on attend de lui).

3. Recherche Dichotomique (Liste triée)

Utilise la stratégie "Diviser pour régner" en coupant la liste en deux à chaque étape.

while i_deb <= i_fin:
    i_cen = (i_deb + i_fin) // 2
    if L[i_cen] == val:
        return True
    elif L[i_cen] < val:
        i_deb = i_cen + 1
    else:
        i_fin = i_cen - 1
Complexité : Comme la taille du problème est divisée par 2 à chaque tour, la complexité est logarithmique, notée O(log(n)).

4. Résumé des Complexités

  • O(1) : Temps constant (pas de boucle).
  • O(log(n)) : Recherche dichotomique (division par 2).
  • O(n) : Recherche linéaire / Parcours simple.
  • O(n²) : Boucles imbriquées.
